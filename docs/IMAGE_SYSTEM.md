# سیستم مدیریت و نمایش تصاویر (Media System)

این مستند حاوی تصمیمات معماری و راه‌حل‌های پیاده‌سازی شده برای سیستم تصاویر پروژه است.

## مشکل تاریخی (Resolved)
در مراحل اولیه پروژه، تصاویر آپلود شده در Liara Object Storage در محیط لوکال با خطای **404 Not Found** مواجه می‌شدند.
**علت:**
1. باکت‌های S3 به صورت پیش‌فرض Private هستند.
2. عدم تطابق دامنه (`.site` vs `.space`) و مشکلات CORS در localhost.
3. حذف امضای امنیتی (Query String) توسط توابع کمکی کلاینت.

## راهکار پیاده‌سازی شده: Media Proxy
برای حل دائمی این مشکل، از الگوی **Proxy** استفاده شده است.

### ۱. پروکسی داخلی (Route Handler)
فایل در مسیر `app/api/media/[...path]/route.ts` وظیفه دارد درخواست‌ها را دریافت کرده و با استفاده از `AWS SDK` و کلیدهای دسترسی سمت سرور، فایل را از لیارا بخواند و به صورت Stream به مرورگر برگرداند.

### ۲. تابع تولید URL
تمام تصاویر باید از طریق تابع `getPublicImageUrl` در `lib/upload-image.ts` پردازش شوند. این تابع آدرس‌ها را به صورت خودکار به فرمت زیر تبدیل می‌کند:
`http://localhost:3000/api/media/products/ID/image.webp`

## قوانین مهم برای توسعه‌دهندگان و ایجنت‌ها:
1. **هرگز** از آدرس مستقیم لیارا (`storage.c2.liara.site/...)` در تگ‌های `<img>` استفاده نکنید.
2. همیشه از تابع `getImageUrl` یا `getPublicImageUrl` استفاده کنید.
3. در صورت تغییر در فایل `.env` حتماً متغیر `NEXT_PUBLIC_LIARA_BUCKET_NAME` را با نام باکت مقداردهی کنید تا کلاینت بداند مسیر پروکسی را چگونه بسازد.
4. اگر تصاویر در سرور اصلی (Production) نمایش داده نشدند، ابتدا تنظیمات CORS در پنل لیارا را بررسی کنید، هرچند سیستم پروکسی فعلی نیاز به تنظیمات خاصی در لیارا ندارد.
